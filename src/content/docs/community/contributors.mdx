# Contributors

OpenJobSpec is built by a growing community of contributors across the world.
Every contribution matters — from fixing a typo to implementing a new backend.

## Maintainers

The project is led by the **Steering Committee** and four Special Interest Groups:

<CardGrid>
  <Card title="SIG-Spec" icon="document">
    Spec editors, RFC reviewers, extension designers
  </Card>
  <Card title="SIG-SDK" icon="puzzle">
    SDK maintainers across 8 programming languages
  </Card>
  <Card title="SIG-Backend" icon="server">
    Backend implementors (Redis, Postgres, NATS, Kafka, SQS, Lite, AMQP)
  </Card>
  <Card title="SIG-Tooling" icon="setting">
    CLI, Admin UI, Playground, K8s Operator, Terraform provider
  </Card>
</CardGrid>

## How to Contribute

We welcome contributions of all kinds:

- 🐛 **Bug reports** — [File an issue](https://github.com/openjobspec/openjobspec/issues/new?template=bug_report.yml)
- 💡 **Feature ideas** — [Start a discussion](https://github.com/openjobspec/openjobspec/discussions/new?category=ideas)
- 📝 **Documentation** — Fix typos, improve guides, add examples
- 🧪 **Testing** — Add conformance tests, property tests, fuzz tests
- 🔧 **Code** — Implement features, fix bugs, improve performance

See our [Contributing Guide](/governance/contributing) for the full path from zero to first PR.

## Good First Issues

New to OJS? Start with our curated [Good First Issues](https://github.com/openjobspec/openjobspec/labels/good%20first%20issue) —
each one includes context, hints, and a mentoring offer from a maintainer.

## Recognition

We use [All Contributors](https://allcontributors.org/) to recognize every type of contribution.
Contributors are listed in each repository's README.
